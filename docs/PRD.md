# PRD — 마블 파티

MVP 캡슐(10줄)
1. 목표: 행사 현장에서 모두가 동시에 참여하는 공정한 제비뽑기 경험을 만든다
2. 페르소나: 오프라인 행사(세미나/파티) 참가자들
3. 핵심 기능(=FEAT-1): 방 생성·QR 입장(빠른 참여 + 대기실 확인)
4. 성공 지표(노스스타): 공정하다고 느낀 비율
5. 입력 지표(1~2개): 불만 제기 건수, 규칙 이해도
6. 비기능 요구(최소 1개): 모바일 웹에서 5초 내 입장 완료(현장 네트워크 기준)
7. Out-of-scope(이번엔 안 함): 룰렛/레이싱 실제 플레이, 장애물 조작, 당첨자 연출
8. Top 리스크 1개: MVP만으로 재미/몰입 검증이 부족함
9. 완화/실험 1개: 짧은 영상/종이 프로토타입으로 공정성·재미 설문
10. 다음 단계(바로 할 행동 1개): 행사 참가자 5명에게 입장 흐름 사용성 테스트

## 문제 정의
- 행사 현장에서 추첨은 필요하지만, 참여자들이 수동적이라 집중이 떨어진다.
- 진행자가 재미를 만들기 어렵고 분위기가 쉽게 가라앉는다.
- 공정하다는 인식이 약하면 신뢰가 깨진다.

## 목표
- 모두가 동시에 “내가 참여했다”고 느끼는 순간을 만든다.
- 규칙 설명 없이도 바로 이해되는 참여 흐름을 제공한다.
- 공정하다는 체감을 최우선으로 확보한다.

## 사용자 페르소나
- 오프라인 행사(세미나/파티) 참가자들: 짧은 시간에 집중하고 즐거움을 얻고 싶음.
- 진행자(보조 페르소나): 빠르게 진행하고 분위기를 살리고 싶음.

## 사용자 스토리(SSOT)
- FEAT-0: 참여자로서, 규칙을 한눈에 이해하고 “준비 완료”로 참여 의사를 표시한다.
- FEAT-1: 진행자로서, 방을 만들고 QR을 공유해 모두가 즉시 들어오게 한다.
- FEAT-1: 참여자로서, QR을 찍고 5초 안에 입장한다.
- FEAT-2: 진행자로서, 룰렛/레이싱을 시작해 흐름을 통제한다.
- FEAT-3: 참여자로서, 버튼을 눌러 장애물을 조작하고 참여감을 느낀다.

## 범위(MVP)
- 포함: FEAT-1 중심의 방 생성·QR 입장·대기실 인원 확인
- 포함(최소): FEAT-0의 규칙 요약/준비 완료 표시
- 제외: FEAT-2, FEAT-3 및 당첨자 연출

## Non-goals
- 복잡한 계정/로그인
- 행사 기록 저장 및 히스토리 제공
- 외부 서비스 연동(결제/로그인/지도)

## 해결 가설
- “모두가 같은 순간에 접속하고 준비를 누르는 경험”만으로도 참여감이 상승할 것이다.

## 성공 지표
- 노스스타: 공정하다고 느낀 비율
- 입력 지표: 불만 제기 건수, 규칙 이해도

## 가정 검증(검증 가능한 형태)
- 가정: “5초 내 입장”이 가능하다 → 검증: 실제 행사장에서 평균 입장 시간 측정
- 가정: 규칙이 즉시 이해된다 → 검증: 1문장 규칙 설명만으로 이해도 조사
- 가정: 공정성 체감이 유지된다 → 검증: 불만 제기 여부 및 짧은 설문

## 실험 → 관측 → 학습 → 다음 가설 루프
- 실험: 5명 소규모 행사에서 입장/대기실만 테스트
- 관측: 입장 성공률, 규칙 이해도, 불만 제기 여부
- 학습: 병목 원인(네트워크/UI/설명) 파악
- 다음 가설: FEAT-2/3가 공정성 체감을 높일지 확인

## 톤 & 말투
- 쇼 진행자 느낌, 과하지 않게: “준비되셨나요?”, “곧 시작합니다”

## 아이디어 검증 질문
- “진행자가 룰렛을 돌리고 참여자가 버튼을 누르는 방식이 재미있을까요?”

## 장애물 아이디어(후순위 참고)
- 기본: 핀볼 플리퍼(타격감, Tap)
- 기본: 시한 폭탄(카오스 유발, Tap)
- 확장: 펀치 피스톤(밀쳐내기, Tap)
- 확장: 강풍기(공중부양, Hold)
- 확장: 함정 문(열림/닫힘, Toggle)
- 공정성 가드레일: “아예 안 떨어지게 막는 동작” 금지
